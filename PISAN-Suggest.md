### AI Integration Enhancements

This is the highest-leverage area for the remaining grade. Pick one of these and ship it, do not try all.

- **Implement the planned NavMesh agent.** Plan.md Features 1-19 are well-thought-out but unimplemented. Even a minimal version (`NavMeshAgent` per evacuee, `NavMeshModifierVolume` for a fire region, `SetAreaCost` to encode hazard aversion) would convert "AI planning doc" into "AI in the application." This is the path the team already designed for.
- **Add an LLM-driven scenario assistant.** Allow a user to type "place a fire near the east stairwell and add 50 agents on floor 2" and have an LLM translate that into sandbox commands (you already have an `ISandboxEditorCommand` and `SandboxCommandHistory`). This is genuinely "AI central to the logic" rather than chat-on-the-side because the LLM is the authoring interface.
- **Add a learned evacuation-time predictor.** Run the simulation many times across procedurally generated layouts, log (layout features -> evac time), train a small regression model, and expose a "predicted evacuation time" indicator that updates live as the user edits the building. This is the most technically defensible AI integration for the rubric language ("predictive modeling, recommendation engines").
- **Crowd behavior policy via simple RL or scripted policies with comparison.** Even a hand-coded set of policies (leader-following, nearest-exit, exit-load-balancing) with a UI toggle would let the user see AI decision-making affect outcomes.

Whichever option is chosen, document it on the `How It Works` page and reference the implementing files so a grader can verify "AI is central" without reading the source.
